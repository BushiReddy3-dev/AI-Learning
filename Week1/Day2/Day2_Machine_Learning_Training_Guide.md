# Part 1: Theory

## 1. What is Machine Learning?

Machine Learning is a subset of Artificial Intelligence that enables systems to learn patterns from data and make predictions without being explicitly programmed.

### Simple Example

Traditional programming:

```text
Rules + Data
     ↓
Program
     ↓
Output
```

Machine Learning:

```text
Data + Expected Output
        ↓
Training
        ↓
Model
        ↓
Prediction
```

### Real-World Examples

- Spam detection
- Fraud detection
- Disease prediction
- House price prediction
- Recommendation systems
- Customer segmentation

---

## 2. Important ML Terminologies

### Data

Data is raw information used to train or test a model.

Example:

```text
Employee age, salary, experience, department, performance score
```

### Dataset

A dataset is a collection of data records.

Example:

| Age | Salary | Experience | Left Company |
|---:|---:|---:|---|
| 25 | 50000 | 2 | No |
| 35 | 70000 | 8 | No |
| 22 | 45000 | 1 | Yes |

### Feature

A feature is an input column used by the model.

Examples:

- Age
- Salary
- Experience
- Department

### Label

A label is the output value the model tries to predict.

Example:

```text
Left Company = Yes or No
```

### Model

A model is the output of the training process. It learns patterns from data and makes predictions.

### Training

Training is the process of teaching a model using historical data.

### Prediction

Prediction is the output generated by the trained model for new input data.

---

## 3. Types of Machine Learning

## 3.1 Supervised Learning

Supervised Learning uses labeled data.

This means both input and expected output are available during training.

### Example

| Email Text | Label |
|---|---|
| Win free money | Spam |
| Meeting at 3 PM | Not Spam |

### Common Use Cases

- Spam detection
- Loan approval
- Fraud detection
- House price prediction
- Medical diagnosis

### Common Algorithms

- Linear Regression
- Logistic Regression
- Decision Tree
- Random Forest
- Support Vector Machine

---

## 3.2 Unsupervised Learning

Unsupervised Learning uses unlabeled data.

The model identifies hidden patterns or groups in the data.

### Example

Customer purchase data without predefined labels.

The model may group customers into:

- Budget customers
- Premium customers
- Frequent buyers
- Occasional buyers

### Common Use Cases

- Customer segmentation
- Pattern discovery
- Market basket analysis
- Document clustering
- Anomaly detection

### Common Algorithms

- K-Means Clustering
- Hierarchical Clustering
- Principal Component Analysis

---

## 3.3 Reinforcement Learning

Reinforcement Learning is based on rewards and penalties.

The system learns by taking actions and receiving feedback.

### Example

A robot learns to move by receiving:

- Reward for correct movement
- Penalty for wrong movement

### Common Use Cases

- Self-driving vehicles
- Robotics
- Game AI
- Drone navigation
- Industrial automation

---

# 4. Training Data and Testing Data

Machine Learning data is usually divided into training and testing parts.

### Training Data

Training data is used to teach the model.

### Testing Data

Testing data is used to evaluate whether the model works well on new data.

### Example Split

```text
Dataset
   │
   ├── Training Data
   └── Testing Data
```

A common split is:

```text
80% Training Data
20% Testing Data
```

---

# 5. Machine Learning Lifecycle

The Machine Learning lifecycle contains the following stages:

```text
Data Collection
      ↓
Data Cleaning
      ↓
Feature Selection
      ↓
Model Training
      ↓
Model Testing
      ↓
Model Deployment
      ↓
Monitoring
```

## Lifecycle Explanation

### Data Collection

Collect data from databases, files, APIs, logs, or business systems.

### Data Cleaning

Remove incorrect, duplicate, missing, or inconsistent data.

### Feature Selection

Choose important input columns for the model.

### Model Training

Train the model using historical data.

### Model Testing

Evaluate the model using test data.

### Model Deployment

Deploy the trained model as an API or service.

### Monitoring

Monitor model accuracy, performance, and data changes.
